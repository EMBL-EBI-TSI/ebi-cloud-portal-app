Single Sign On

EBI Cloud Portal can be accessed using Single Sign On with LS associated account or EMBL account. LS has associated with different Identity providers like Google, ORCID, LinkedIn, institutional accounts, etc. If you have an account with any of the IdPs, you can proceed with registering with LS, the Single Sign On process involves email verification, and finishes with a "We've got your registration" message from LS.

When you first log in to the EBI Cloud Portal, you will first be asked if you allow your home institution (or Google, etc) to release your data to LS, where you can choose to allow/disallow the release of your data to the EBI Authentication Gateway. EMBL SSO involves just asking to login EMBL account.

Creating a Configuration, from scratch

A Configuration is a way of reusing and sharing cloud credentials and parameters for Applications. In order to be able to deploy an Application in a cloud resource, we need to provide cloud credentials (Cloud Provider parameters) and possibly additional parameters (Deployment Parameters) that are specific to the cloud resource (e.g. network IDs, etc).

It can be a tedious process to introduce these parameters each time we want to deploy an Application. Moreover, some of these are technical details that can be provided by the user managing the cloud resource we are going to deploy to. For that reason, the EBI Cloud Portal introduces the concept of Configuration as a way of encapsulating all that information so it is easy to define, reuse, and share.

Configurations can be shared with you by other users, but you can also create your own. Let's see how. Click on , a side navigation bar appears. Click on 'Profile' which will then take you to profile page where you can add, share and remove Cloud Providers, Configurations, and Deployment Parameters.
